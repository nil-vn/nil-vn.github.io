When a customer claims that some deliverables have not been met close to the end of a project, the project manager should take the following steps: A. Refer to the requirements traceability matrix: The project manager should first review the project's requirements traceability matrix to confirm if the customer's claim is valid. The requirements traceability matrix shows the link between the project's requirements, project scope, and deliverables. By reviewing the matrix, the project manager can identify any missing deliverables or requirements that may have been overlooked.
