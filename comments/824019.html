When new or conflicting requirements are discovered during Project Scope Management, and they differ from what is documented in the project charter, the project manager must follow the formal change control process. The project charter defines the original high-level scope. Any change to scope—especially if it deviates from what was initially approved—must go through integrated change control. This means documenting the change, assessing its impact, and submitting it to the Change Control Board (CCB) for approval before implementation. The configuration management plan is part of the project management plan that outlines how changes will be managed.
